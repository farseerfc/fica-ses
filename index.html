Machine Learning Algorithm

  1. Tokenization
  2. Divide into N-grams
  3. Calculate \(tf-idf\) vector for grams in clonesets
  4. Calculate similarity between each pair of clonesets in training-set and evaluation-set based on their \(tf-idf\) vector.
  5. Calculate the possibilities that a cloneset from evaluation-set should belongs to interesting group or un-interesting group.
  6. Make prediction based on the above possibilities.
  7. Sort the prediction result and present to user.

{
	struct object *obj = lookup_object(sha1);

	if (!obj->type)
		obj->type = OBJ_BLOB;
	if (obj->type != OBJ_BLOB) {

}

\(tf(t,d)\) represent the importance of token n-gram \(t\) in a cloneset \(d\).

Inverse Document Frequency

\[ idf(t,D) = \log\frac{|D|}{1 + |{d \in D : t \in d}|} \]

\(idf(t,D)\) represents the inversed importance of token n-gram \(t\) in all the clonesets \(D\).

\[ tfidf(t, d, D) = tf(t, d) \times idf(t, D) \]

\(tfidf(t,d,D)\) together represents the importance of of token n-gram \(t\) in a cloneset \(d\) among all the clonesets \(D\).

\[ \overrightarrow{tfidf(d, D)} = \left[ tfidf(t,d,D) \quad \forall t \in d \right] \]

Vector \(\overrightarrow{tfidf(d, D)}\) contains the importance of all the token n-gram in \(d\).

Normalized Similarity

\[ sim(a, b, D) = \overrightarrow{tfidf(a, D)} \cdot \overrightarrow{tfidf(b, D)} \\ nsim(a, b, D) = \left\{ \begin{array}{c} 0 , sim(a, b, D) = 0 \\ \frac{sim(a, b, D)}{|\overrightarrow{tfidf(a, D)}| \cdot |\overrightarrow{tfidf(b, D)}| } , \mbox{otherwise} \end{array} \right. \]

This calculates normalized similarity between two clonesets.

For example, if the \(tfidf\) vector of two clonesets \(a\) and \(b\) are :


tfidf(a,D): { "A": 0.9, "B":0.4, "C":0.6, "D":0.5}
tfidf(b,D): { "B": 0.8, "C":0.7, "E":0.3, "F":0.8}

\[ |\overrightarrow{tfidf(a, D)}| = \sqrt{0.9^2+0.4^2+0.6^2+0.5^2} = 1.26 \\ |\overrightarrow{tfidf(b, D)}| = \sqrt{0.8^2+0.7^2+0.3^2+0.8^2} = 1.36 \\ sim(a,b,D) = [0.9\ 0.4\ 0.6\ 0.5\ 0.0\ 0.0] \cdot [0.0\ 0.8\ 0.7\ 0.0\ 0.3\ 0.8] \\ = 0.9\times 0.0+0.4\times 0.8+0.6\times 0.7+0.5\times 0.0+0.0\times 0.3+0.0\times 0.8 \\ = 0.74 \\ nsim(a,b,D) = \frac{0.74}{1.26 \times 1.36} \\ = 0.43 \]

Possibility

a cloneset \(d\) belong to a group \(M\)

\[ poss(d, M) = \left\{ \begin{array}{cl} 1 &, |M| = 0 \\ \frac{\sum_{\forall m \in M} nsim(d, m, M) }{|M|} &, \mbox{otherwise} \end{array} \right. \]

Conclusion

  • We have shown that users of CDT may have different opinions on whether a code clone is useful or interesting.
  • We proposed a method to predict the interesting-ness based on machine learning algorithms.
  • We make use of a virtualization called Force-Directed Graph to illustrate similarity among clonesets.
  • We developed a web-based system called Fica to aid the machine learning process.
  • The Fica system showed more than 70% accuracy in average and more than 90% accuracy in particular user and project.
